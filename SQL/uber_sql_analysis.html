Q 01
Overall Trip Status Distribution
Baseline health · How requests resolve across all 6,745 rides
SQL — DuckDB
SELECT Status, COUNT(*) AS total_requests, ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER(), 2) AS pct_share, SUM(COUNT(*)) OVER() AS grand_total FROM rides GROUP BY Status ORDER BY total_requests DESC;
StatusTotal RequestsPct Share %Grand Total
Trip Completed2,83141.97%6,745
No Cars Available2,65039.29%6,745
Cancelled1,26418.74%6,745
Insights Drawn
  • Only 41.97% of all ride requests result in a completed trip — fewer than 1 in 2 requests reaches its destination.
  • The combined failure rate is 58.03%. Of this, supply-side failure (No Cars) accounts for 39.29% — making it the single largest source of unmet demand, nearly double the cancellation rate.
  • Cancellations at 18.74% represent a demand-side or driver-acceptance problem — the car existed but the trip still didn't happen, pointing to pricing, ETA, or driver refusal issues.
  • This 3-way split sets the benchmark: any operational improvement must address supply gap first, then cancellation behavior, to meaningfully lift the 42% baseline.
Q 02
Demand & Outcome Breakdown by Pickup Location
Airport vs City · Volume, completion, and failure mode comparison
SQL — DuckDB
SELECT "Pickup point", COUNT(*) AS total_requests, SUM(CASE WHEN Status = 'Trip Completed' THEN 1 ELSE 0 END) AS completed, SUM(CASE WHEN Status = 'No Cars Available' THEN 1 ELSE 0 END) AS no_cars, SUM(CASE WHEN Status = 'Cancelled' THEN 1 ELSE 0 END) AS cancelled, ROUND(SUM(CASE WHEN Status = 'Trip Completed' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 1) AS completion_rate_pct, ROUND(SUM(CASE WHEN Status = 'No Cars Available' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 1) AS no_car_rate_pct FROM rides GROUP BY "Pickup point" ORDER BY total_requests DESC;
Pickup PointTotal RequestsCompletedNo CarsCancelledCompletion Rate %No Car Rate %
City 3,507 1,504 937 1,066 42.9% 26.7%
Airport 3,238 1,327 1,713 198 41.0% 52.9%
Insights Drawn
  • Airport's No-Car rate of 52.9% means over half of all Airport pickup requests receive no driver — a structural supply crisis, not a random spike.
  • City's primary failure mode is cancellations (1,066 cases, 30.4%) vs Airport's cancellations (198, 6.1%). City drivers cancel 5× more often than Airport drivers, suggesting route preference or pricing friction.
  • Despite higher no-car failures, Airport completion rate (41.0%) is nearly on par with City (42.9%) — when a driver is available at Airport, the ride almost always completes.
  • The two locations require entirely different operational fixes: Airport needs supply injection, City needs cancellation root-cause analysis.

Q 07
Trip Duration Statistics by Pickup Point
Operational efficiency · Full distribution stats on completed journey times
SQL — DuckDB
SELECT "Pickup point", ROUND(AVG("Trip duration (mins)"), 2) AS avg_duration_mins, ROUND(MIN("Trip duration (mins)"), 2) AS min_duration_mins, ROUND(MAX("Trip duration (mins)"), 2) AS max_duration_mins, ROUND(STDDEV("Trip duration (mins)"), 2) AS std_dev_mins, ROUND(PERCENTILE_CONT(0.25) WITHIN GROUP (ORDER BY "Trip duration (mins)"), 2) AS q1_mins, ROUND(PERCENTILE_CONT(0.50) WITHIN GROUP (ORDER BY "Trip duration (mins)"), 2) AS median_mins, ROUND(PERCENTILE_CONT(0.75) WITHIN GROUP (ORDER BY "Trip duration (mins)"), 2) AS q3_mins, COUNT(*) AS completed_trips FROM rides WHERE "Trip duration (mins)" IS NOT NULL GROUP BY "Pickup point";
Pickup PointAvg DurationMinMaxStd DevQ1MedianQ3Completed Trips
Airport 52.24 min 21.18 min 82.07 min 13.85 41.0 min 51.6 min 64.0 min 1,327
City 52.57 min 20.78 min 83.00 min 13.86 41.0 min 53.0 min 63.88 min 1,504
Insights Drawn
  • Airport and City have nearly identical trip duration profiles — same average (~52 min), same IQR (41–64 min), same standard deviation (13.85–13.86). This indicates a single shared corridor (likely Airport ↔ City), not independent route networks.
  • Standard deviation of ~14 minutes relative to a 52-minute mean is high (27% CoV), meaning trip times are highly variable. A rider could experience anything from 21 to 83 minutes — hard to predict or plan around.
  • The IQR of 23 minutes (Q1=41, Q3=64) means the middle 50% of trips fall within a 23-minute range — acceptable for airport transfers but tight for scheduling purposes.
  • The matching distribution confirms that once a driver is assigned, the service quality is consistent regardless of pickup location. All performance gaps stem from availability failure, not execution failure.
Q 08
Critical Failure Zones — Hour × Pickup With >65% Total Failure Rate
CTE-based triage · Isolates every slot where combined no-cars + cancellations exceed 65%
SQL — DuckDB (CTE)
WITH peak_hours AS ( SELECT "Request hour" AS hour, "Pickup point" AS pickup, COUNT(*) AS total, SUM(CASE WHEN Status = 'No Cars Available' THEN 1 ELSE 0 END) AS no_cars, SUM(CASE WHEN Status = 'Cancelled' THEN 1 ELSE 0 END) AS cancelled, ROUND(SUM(CASE WHEN Status = 'No Cars Available' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 1) AS no_car_pct, ROUND(SUM(CASE WHEN Status = 'Cancelled' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 1) AS cancel_pct, ROUND((SUM(CASE WHEN Status = 'No Cars Available' THEN 1 ELSE 0 END) + SUM(CASE WHEN Status = 'Cancelled' THEN 1 ELSE 0 END)) * 100.0 / COUNT(*), 1) AS total_failure_pct FROM rides GROUP BY "Request hour", "Pickup point" ) SELECT hour, pickup, total, no_cars, cancelled, no_car_pct, cancel_pct, total_failure_pct FROM peak_hours WHERE total_failure_pct > 65 ORDER BY total_failure_pct DESC;
